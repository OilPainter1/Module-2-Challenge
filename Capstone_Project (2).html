# Overview:
# A bank makes loans and receives profits on those loans as a result of interests.
# When a customer pays back there loan plus interest, the bank makes money
# The problem is when a customer can not pay back their loan, that is, they default on 
# the loan.
# The data set given about the customers applying for the loan is what we will be
# analyzing and building models off of.
# The data has various information about the customer's credit, their financial
# situation, and whether or not they were able to pay back the loan
# This is a classification problem and therefore will involve classification models
# During this analysis and when building models, it will be most important to detect on 
# those likely to default on their loans. Therefore we want the model to predict that a 
# customer will default on a loan when they actually did.
# Recall is the measure of this detection and therefore will be the most important
# way to determine a models success and use.
# Through the use of different models we will see what drives loan defaulting
# and what banks should consider when granting loans.

def update_outliers(x,**kwargs):
    if x>kwargs["outlier_threshold"]:
        x=kwargs["median"]
        return x
    else:
        return x

# Conclusion:
# The best model in terms of recall score for loan defaults is the K-Nearest Neighbor
# model with k=1 and p=1. Not only did it have the highest recall score of 80 percent
# but also had the highest accuracy at 95 percent. Its predictions are correct 95 percent 
# of the time and will detect a probable loan defaulter 80 percent of the time. In general,
# a customer who has feature values which are close to feature values of loan defaulters are
# likely to be loan defaulters. A DEBTINC value in the range of 35-40 along with a DELINQ
# value of 0.0 should probably not be given a loan. The K-Nearest Neighbor algorithm with
# k=1 and p=1 also has a high accuracy meaning it will often correctly identify those who 
# will repay the loan and therefore the bank will make a profit. The model best suited for 
# this setting is the K-Nearest Neighbor model with k=1 and p=1.
